# Wonderful-Color

[English](./README.md) | [中文](./README_zh.md)

## 项目简介

Wonderful-Color 是一个轻量级的网页工具，用于生成和探索颜色组合。项目参考了 Coolors 等配色工具，重点在于通过实际开发来学习现代前端技术，并实现具有一定实用价值的功能。

该项目强调简洁的设计、即时反馈的交互，以及基于规则的配色生成逻辑。

## 功能特性

- 支持点击按钮或按空格键生成新的配色方案
- 支持锁定指定颜色，在刷新时保持不变
- 双生成模式并存：
  - `智能模式`：多策略生成候选配色并进行评分选优
  - `渐变模式`：延续经典渐变逻辑并支持锁色分段插值
- 提供风格档位（`自动混合`、`柔和高级`、`鲜艳活力`）
- 提供“强对比优先”选项，增强颜色区分度
- 锁色约束优化：
  - 锁定位置作为硬约束不被修改
  - 未锁定位置围绕锁色进行优化
- 基于对比度自动选择黑/白文字色，保证可读性
- 全屏响应式布局，使用竖向色条展示
- 实时显示颜色的十六进制值
- URL Hash 同步，支持分享与恢复当前配色

## 实现原理

### 配色生成逻辑

项目目前包含两套并存的配色引擎：

- 智能模式：
  - 基于多种和谐策略生成候选配色：
    - analogous（类似色）
    - complementary（互补色）
    - split-complementary（分裂互补）
    - triadic（三角色）
    - tetradic（四色系）
    - monochrome（单色系）
    - hybrid（混合策略）
  - 对每个候选方案进行评分，评分维度包括：
    - 颜色最小距离
    - 色相分布均衡
    - 明度与饱和度层次
    - 文字可读性惩罚
  - 选择总分最高的候选作为输出

- 渐变模式：
  - 无锁色时：从“互补端点”或“两端随机远距离颜色”生成平滑渐变
  - 有锁色时：以锁定颜色为锚点，在段内进行 LCH 插值补全未锁定色块
  - 在输出前再次应用锁色约束，并对未锁定色位做优化

两种模式都使用 `chroma.scale(...).mode('lch')`，以获得更自然的感知过渡。

### 文字对比度处理

为了保证文字在不同背景下的可读性，项目使用对比度比较法：
- 计算背景与黑色文字的对比度
- 计算背景与白色文字的对比度
- 选择对比度更高的文字颜色


## 使用方法

1. 在浏览器中打开页面
2. 按下空格键或点击 “Refresh Colors” 按钮生成新配色
3. 点击锁图标可以锁定当前颜色
4. 再次生成配色时，已锁定颜色不会改变
